# コードレビュー

以下の観点でコードをレビューします。

## セキュリティ
- 認証・認可の適切な実装
- SQL インジェクション・XSS の可能性
- シークレット情報のハードコーディング

## 品質
- 関数・変数名の明確さ
- 重複コードの有無
- エラーハンドリングの適切さ

## パフォーマンス
- 不必要なループ・N+1 問題
- キャッシュの活用可否

各問題は重要度別（Critical / Warning / Suggestion）に分類して報告してください。

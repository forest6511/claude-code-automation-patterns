# 実装前チェックリスト

**実装タスク**: $ARGUMENTS

以下の5項目を検証し、それぞれ PASS / FAIL / SKIP で答えてください。

## C1: 重複実装チェック
- `grep -r "関連クラス名\|関連関数名" --include="*.{ts,py,go}" src/` を実行
- 同等の機能が既存コードに存在しないか確認
- 既存コードを拡張できる場合は FAIL（新規実装不要）

## C2: アーキテクチャ準拠チェック
- `CLAUDE.md` のアーキテクチャ制約を参照
- 依存方向・レイヤー分離・命名規則に違反しないか確認

## C3: 公式ドキュメント確認チェック
- 使用するライブラリ・フレームワークの公式ドキュメントを確認
- 使用予定の API が現行バージョンで有効か確認

## C4: OSS 参照チェック
- 同様のパターンを実装している OSS が存在するか確認
- 車輪の再発明を避けられるか評価

## C5: 根本原因特定チェック
- バグ修正・性能改善の場合: 根本原因を特定したか
- 機能追加の場合: ユーザーの真のニーズを確認したか

---

## スコア計算
- PASS 数: N / 5
- 90% 以上（4 / 5 以上）→ 実装開始 OK
- 60% 以下（2 / 5 以下）→ 実装停止・再調査

FAIL / SKIP の項目について、理由と対処策を列挙してください。
